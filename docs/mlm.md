# MLM (Multi-Level Marketing) 시스템 사용 매뉴얼

## 🎯 시스템 개요 및 컨셉

### MLM이란?
**MLM(Multi-Level Marketing)**은 다단계 마케팅 시스템으로, 파트너들이 계층적 네트워크를 구성하여 서로 협력하며 수익을 창출하는 비즈니스 모델입니다.

### 🏗️ 기본 구조
```
레벨 0 (루트)    김대표 (창립자)
              ├── 레벨 1: 이부장, 박과장, 최팀장
              ├── 레벨 2: 정대리, 한사원, 윤인턴
              └── 레벨 3: 조신입, 문인턴...
```

### 💡 핵심 컨셉
1. **계층 구조**: 루트(레벨 0)부터 시작해서 숫자가 클수록 하위 계층
2. **파트너 모집**: 상위 파트너가 하위 파트너를 모집하여 네트워크 확장
3. **커미션 분배**: 매출 발생 시 관련된 상위 파트너들에게 커미션 자동 분배
4. **티어 시스템**: 성과에 따라 브론즈 → 실버 → 골드 → 플래티넘 → 다이아몬드로 승급

### 🎁 수익 구조
- **직접 판매 커미션**: 본인이 판매한 경우
- **팀 보너스**: 하위 파트너들의 매출에서 일정 비율
- **관리 보너스**: 팀 관리에 대한 추가 보상
- **모집 보너스**: 신규 파트너 영입 시 일회성 보상
- **등급 보너스**: 티어 승급 시 특별 보상

## 🚀 관리자 메뉴 접속 및 사용법

### 📋 메뉴 구성
관리자 로그인 후 좌측 사이드바에서 **"네트워크 관리"** 섹션을 확인할 수 있습니다:

```
📊 네트워크 관리
├── 🌲 네트워크 트리          (/admin/partner/network/tree)
├── 👥 파트너 모집           (/admin/partner/network/recruitment)
├── 💰 커미션 관리           (/admin/partner/network/commission)
├── 📈 네트워크 분석         (하위 메뉴)
│   ├── 📊 분석 대시보드      (/admin/partner/network/analytics/dashboard)
│   ├── 🎯 성과 분석         (/admin/partner/network/analytics/performance)
│   └── 🧬 계보 분석         (/admin/partner/network/analytics/genealogy)
└── ⚙️ 계층 관리             (/admin/partner/network/hierarchy/management)
```

## 📖 단계별 사용 가이드

### 1️⃣ 네트워크 트리 보기

**접속**: `/admin/partner/network/tree`

**화면 설명**:
- 상단에 전체 파트너 수, 활성 파트너 수, 총 매출, 평균 팀 크기 통계 표시
- 중앙에 트리 형태로 파트너 계층 구조 시각화
- 각 파트너 카드에는 이름, 티어, 매출, 하위 파트너 수, 성과 점수 표시
- 우측에 레벨별 분포, 티어별 분포, 모집 활동 통계

**주요 기능**:
- 🔍 **트리 옵션**: 최대 깊이 설정, 루트 파트너 선택, 비활성 파트너 포함 여부
- 📈 **성과 확인**: 각 파트너의 월 매출, 팀 매출, 커미션, 팀 크기 한눈에 보기
- 👥 **직접 모집**: 파트너 카드에서 "모집" 버튼 클릭하여 즉시 하위 파트너 영입

**사용 방법**:
1. "트리 옵션" 버튼으로 표시 설정 조정
2. 파트너 카드 클릭하여 상세 정보 확인
3. "모집" 버튼으로 신규 파트너 영입
4. "전체 확장/접기" 버튼으로 트리 뷰 조절

### 2️⃣ 파트너 모집 관리

**접속**: `/admin/partner/network/recruitment`

**화면 설명**:
- 상단에 총 관계 수, 활성 관계 수, 이번 달 모집 수, 성공률 통계
- 최고 모집자들의 실적 순위
- 하단에 모든 모집 관계 목록 (상위-하위 파트너, 모집자, 모집일, 성과)

**주요 기능**:
- ➕ **파트너 모집**: "파트너 모집" 버튼으로 새로운 관계 생성
- 🔍 **필터링**: 상태별, 모집자별 필터링 가능
- ❌ **관계 해제**: 문제가 있는 관계 해제 및 사유 기록
- 📊 **성과 추적**: 각 모집 관계의 매출 성과 모니터링

**모집 프로세스**:
1. "파트너 모집" 버튼 클릭
2. 상위 파트너 선택 (모집을 받을 파트너)
3. 하위 파트너 선택 (새로 영입할 파트너)
4. 모집자 선택 (실제 모집 활동한 사람, 선택사항)
5. 모집 메모 작성 후 "모집하기" 실행

### 3️⃣ 커미션 관리

**접속**: `/admin/partner/network/commission`

**화면 설명**:
- 상단에 총 커미션, 지급 완료 금액, 미지급 금액, 평균 커미션 통계
- 커미션 타입별 통계 (직접 판매, 팀 보너스, 관리 보너스 등)
- 하단에 모든 커미션 내역 (파트너, 금액, 상태, 발생일)

**주요 기능**:
- 🧮 **커미션 계산**: "커미션 계산" 버튼으로 매출 발생 시 커미션 자동 계산
- 💳 **지급 처리**: 개별 또는 대량 커미션 지급 처리
- 📊 **통계 확인**: 타입별, 기간별 커미션 분석
- 🔍 **필터링**: 상태, 타입, 파트너, 기간별 필터링

**커미션 계산 방법**:
1. "커미션 계산" 버튼 클릭
2. 매출을 발생시킨 파트너 선택
3. 매출 금액 입력 (예: 1,000,000원)
4. 주문 정보 입력 (선택사항, JSON 형태)
5. "계산하기" 실행하면 자동으로 상위 파트너들에게 커미션 분배

### 4️⃣ 분석 대시보드

**접속**: `/admin/partner/network/analytics/dashboard`

**화면 설명**:
- 핵심 지표: 총 파트너 수, 월간 총 매출, 지급된 커미션, 활성 네트워크 관계
- 네트워크 성장 추이 차트 (7일/30일/90일 단위)
- 티어별 분포 원형 차트
- 최고 성과자 순위, 최근 활동 내역
- 커미션 타입별 현황

**활용법**:
- 📈 **성장 추세 확인**: 기간별 버튼으로 성장 패턴 분석
- 🏆 **우수 파트너 파악**: 성과 순위로 인센티브 대상 선정
- 📊 **균형 상태 점검**: 티어별 분포로 네트워크 건강도 확인

### 5️⃣ 성과 분석

**접속**: `/admin/partner/network/analytics/performance`

**화면 설명**:
- 평균 월 매출, 평균 팀 크기, 평균 커미션, 활동률, 관계 유지율, 커미션 비율
- 티어별 성과 비교 차트 (매출/커미션/팀 크기별 전환 가능)
- 성과 분포 원형 차트 (상위 10%, 상위 30%, 중간 40%, 하위 20%)
- 성과 순위 테이블 (20위까지, 정렬 기준 변경 가능)

**활용법**:
- 🎯 **성과 기준 설정**: 각 티어별 평균 성과 확인하여 목표 설정
- 📊 **불균형 감지**: 성과 분포로 네트워크 건강도 진단
- 🏅 **우수자 선별**: 다양한 기준으로 정렬하여 포상 대상 선정

### 6️⃣ 계보 분석

**접속**: `/admin/partner/network/analytics/genealogy`

**화면 설명**:
- 창립 파트너 수, 최대 깊이, 최대 계보 크기, 활성 계보 수
- 주요 계보 현황 (가문별 구성원 수)
- 계층 깊이 분석 차트
- 세대별 분석 테이블, 계보 성공 사례

**활용법**:
- 🧬 **계보 추적**: "계보 검색" 버튼으로 특정 파트너의 상하위 관계 조회
- 📈 **가문 성과**: 주요 계보별 성장률 및 성과 비교
- 🎯 **전략 수립**: 성공 사례 분석으로 성장 전략 도출

### 7️⃣ 계층 관리

**접속**: `/admin/partner/network/hierarchy/management`

**화면 설명**:
- 총 계층 수, 최상위 파트너 수, 연결 끊긴 파트너 수, 균형 점수
- 레벨별 파트너 현황 (각 레벨의 파트너 카드 형태로 표시)
- 우측에 빠른 작업 버튼들과 최적화 제안

**주요 기능**:
- 🔄 **파트너 이동**: 문제가 있는 구조 수정을 위한 파트너 이동
- ⚖️ **구조 재편**: 전체적인 네트워크 균형 조정
- 🔍 **문제 탐지**: 고아 노드, 불균형 노드 자동 탐지
- ⚡ **최적화**: 구조 최적화 및 균형 맞추기

**파트너 이동 방법**:
1. "파트너 이동" 버튼 클릭
2. 이동할 파트너 선택
3. 새로운 상위 파트너 선택
4. 이동 사유 작성 후 실행

## ⚠️ 주의사항 및 모범 사례

### 🚫 주의사항
1. **순환 관계 금지**: A → B → A 형태의 순환 구조는 시스템에서 자동 차단
2. **모집 한도**: 각 티어별로 관리 가능한 최대 하위 파트너 수 제한 존재
3. **커미션 중복**: 동일한 매출에 대해 중복 커미션 계산 불가
4. **구조 변경**: 파트너 이동이나 구조 재편은 신중하게 결정 (되돌리기 어려움)

### ✅ 모범 사례
1. **정기 모니터링**: 주간 단위로 분석 대시보드 확인
2. **균형 유지**: 한 파트너에게 과도한 하위 구성원이 몰리지 않도록 관리
3. **성과 인정**: 우수 성과자에게 적절한 보상 및 승급 기회 제공
4. **투명성**: 커미션 지급 내역을 정기적으로 공개하여 신뢰 구축

## 💰 실제 사용 예시

### 📊 시나리오 1: 신규 파트너 모집

**상황**: 김대표(레벨 0)가 이부장(신규)을 영입하려고 함

**단계**:
1. `/admin/partner/network/recruitment` 접속
2. "파트너 모집" 버튼 클릭
3. 상위 파트너: "김대표" 선택
4. 하위 파트너: "이부장" 선택
5. 모집 메모: "영업 경험 10년, 네트워크 구축 능력 우수"
6. "모집하기" 실행

**결과**:
- 이부장이 레벨 1로 자동 설정
- 김대표의 children_count 증가
- 모집 보너스 자동 지급 (티어별 설정에 따라)

### 💸 시나리오 2: 매출 발생 시 커미션 분배

**상황**: 정대리(레벨 2)가 100만원 매출 달성

**단계**:
1. `/admin/partner/network/commission` 접속
2. "커미션 계산" 버튼 클릭
3. 소스 파트너: "정대리" 선택
4. 매출 금액: "1,000,000" 입력
5. 주문 정보: `{"product": "서비스A", "customer": "고객123"}`
6. "계산하기" 실행

**자동 분배 결과**:
- 정대리 (직접 판매): 100만원 × 5% = 5만원
- 이부장 (1단계 상위): 100만원 × 3% = 3만원
- 김대표 (2단계 상위): 100만원 × 2% = 2만원

### 🔍 시나리오 3: 성과 분석 및 관리

**상황**: 월말 정산 및 성과 평가

**단계**:
1. `/admin/partner/network/analytics/dashboard` 접속
2. 전체 현황 확인: 총 파트너 150명, 월 매출 5억원
3. `/admin/partner/network/analytics/performance` 이동
4. 성과 순위에서 상위 10명 확인
5. 티어별 성과 비교로 승급 대상자 선별

**활용 결과**:
- 상위 10명에게 인센티브 지급 결정
- 승급 조건 충족자 3명 골드 티어로 승급
- 저성과 그룹 대상 교육 프로그램 계획

## ❓ 자주 묻는 질문 (FAQ)

### Q1: 파트너가 여러 상위 파트너를 가질 수 있나요?
**A**: 아니요. 각 파트너는 반드시 하나의 상위 파트너만 가질 수 있습니다. 이는 명확한 계층 구조와 커미션 분배를 위함입니다.

### Q2: 커미션 계산은 언제 이루어지나요?
**A**: 매출이 발생한 즉시 자동으로 계산됩니다. 관리자가 수동으로 "커미션 계산" 기능을 실행하면 해당 매출에 대한 커미션이 상위 파트너들에게 분배됩니다.

### Q3: 파트너를 다른 상위로 이동시킬 수 있나요?
**A**: 네, 가능합니다. `/admin/partner/network/hierarchy/management`에서 "파트너 이동" 기능을 사용하면 됩니다. 단, 이동 시 하위 네트워크 전체가 함께 이동하므로 신중하게 결정해야 합니다.

### Q4: 최대 몇 단계까지 계층을 만들 수 있나요?
**A**: 시스템상 제한은 없지만, 관리 효율성을 위해 보통 5-7단계 정도를 권장합니다. 너무 깊은 계층은 관리가 어렵고 커미션 분배가 복잡해집니다.

### Q5: 티어별 모집 제한은 어떻게 설정하나요?
**A**: `/admin/partner/tiers`에서 각 티어의 `max_children` 값을 설정할 수 있습니다. 예: 브론즈 5명, 실버 10명, 골드 20명 등.

### Q6: 커미션 지급은 어떻게 처리하나요?
**A**: `/admin/partner/network/commission`에서 개별 또는 대량으로 지급 처리할 수 있습니다. "지급 완료" 상태로 변경하면 실제 계좌 이체는 별도 시스템에서 처리합니다.

### Q7: 네트워크가 불균형할 때는 어떻게 하나요?
**A**: `/admin/partner/network/hierarchy/management`의 "구조 최적화" 기능을 사용하거나, 수동으로 파트너를 이동시켜 균형을 맞춥니다.

### Q8: 파트너가 탈퇴하면 하위 네트워크는 어떻게 되나요?
**A**: 해당 파트너의 상위로 하위 파트너들이 자동 이동하거나, 관리자가 수동으로 재배치할 수 있습니다. 탈퇴 전에 네트워크 재편 계획을 세우는 것이 좋습니다.

## 🛠️ 문제 해결 가이드

### 🚨 일반적인 문제 상황

#### 문제 1: "순환 관계가 생성됩니다" 오류
**상황**: 파트너 모집 시 오류 발생
**원인**: A→B→A 형태의 순환 구조 시도
**해결**: 다른 상위 파트너를 선택하거나 네트워크 구조 재검토

#### 문제 2: "관리 가능한 최대 하위 파트너 수에 도달했습니다"
**상황**: 모집 시도 시 거부
**원인**: 해당 파트너의 티어별 모집 한도 초과
**해결**: 파트너 티어 승급 또는 다른 상위 파트너에게 모집

#### 문제 3: 커미션이 계산되지 않음
**상황**: 매출 입력 후 커미션 미발생
**원인**: 상위 파트너 관계가 설정되지 않음
**해결**: 네트워크 트리에서 관계 확인 후 재설정

#### 문제 4: 성능 저하 (페이지 로딩 느림)
**상황**: 대용량 네트워크에서 화면 로딩 지연
**원인**: 과도한 데이터 조회
**해결**: 트리 뷰 깊이 제한, 필터링 활용

### 🔧 시스템 점검 방법

1. **네트워크 무결성 확인**
   - `/admin/partner/network/hierarchy/management` → "계층 검증" 실행
   - 고아 노드, 순환 관계, 불일치 데이터 자동 탐지

2. **커미션 정합성 확인**
   - `/admin/partner/network/commission` → 필터로 기간별 조회
   - 총 매출 대비 커미션 비율 적정성 확인

3. **성과 모니터링**
   - `/admin/partner/network/analytics/dashboard` 정기 확인
   - 비정상적인 급증/급감 패턴 감지
