# 파트너 수익율 분배 시스템 분석 및 기획서

## 📋 개요

현재 파트너 시스템은 3단계 계층 구조로 수익율을 관리하고 있습니다:
1. **Partner Types** (파트너 타입별 기본 설정)
2. **Partner Tiers** (등급별 기본 설정)
3. **Partner Users** (개별 파트너별 개인화된 설정)

## 🔍 현재 시스템 구조 분석

### 1. Partner Types 테이블 (기본 타입별 설정)

#### 수익 관련 컬럼

##### 1. `target_sales_amount` (decimal 15,2) - **목표 매출액**
- **정의**: 각 파트너 타입별로 기대하는 월간 매출 목표
- **용도**: 성과 평가 및 등급 승급의 기준
- **예시**: 세일즈 파트너는 월 500만원, 기술지원 파트너는 월 200만원

##### 2. `target_support_cases` (integer) - **목표 지원 건수**
- **정의**: 각 파트너 타입별로 처리해야 하는 월간 지원/서비스 건수
- **용도**: 매출과 함께 성과 측정의 핵심 지표
- **예시**: 기술지원 파트너는 월 100건, 고객서비스 파트너는 월 120건

##### 3. `commission_bonus_rate` (decimal 5,2) - **타입별 추가 수수료율 (%)**

**🎯 기본 개념:**
- 파트너 타입의 **전문성과 업무 난이도**를 반영한 추가 보너스율
- 등급별 기본 수수료율에 **추가로 더해지는** 보너스
- 파트너의 전문 분야별 **차별화된 보상** 시스템

**⚠️ 중요: 적용 조건에 대한 두 가지 해석**

현재 시스템에서 `target_sales_amount`(매출 목표)와 `commission_bonus_rate`(추가 수수료율)의 관계가 명확하지 않습니다. 다음 두 가지 방식으로 해석 가능합니다:

**🔥 방식 1: 목표 달성 시 추가 보너스 (성과 연동형)**
```
IF (실제 매출 >= target_sales_amount) THEN
    최종 수수료율 = 등급 기본율 + 타입 보너스율 + 개인 조정율
ELSE
    최종 수수료율 = 등급 기본율 + 개인 조정율
END IF
```

**💎 방식 2: 타입별 고정 보너스 (전문성 인정형)**
```
최종 수수료율 = 등급 기본율 + 타입 보너스율 + 개인 조정율
(매출 목표 달성 여부와 무관하게 항상 적용)
```

**📊 각 방식의 특징 비교:**

| 구분 | 성과 연동형 | 전문성 인정형 |
|------|------------|---------------|
| **적용 조건** | 목표 매출 달성 시만 | 항상 적용 |
| **동기 부여** | 높음 (목표 달성 인센티브) | 중간 (안정적 보상) |
| **위험도** | 높음 (미달성시 패널티) | 낮음 (예측 가능) |
| **관리 복잡도** | 높음 (월별 성과 추적) | 낮음 (고정 적용) |
| **적합한 경우** | 성과 중심 조직 | 전문성 중심 조직 |

**🔍 타입별 보너스율의 의미와 근거:**

| 타입 | 보너스율 | 적용 근거 | 주요 업무 |
|------|----------|-----------|----------|
| **컨설턴트 파트너** | **+3.0%** | • 최고 수준의 전문성 요구<br>• 기업 전략 컨설팅 역량<br>• 고액 프로젝트 담당 | 비즈니스 전략 수립, ROI 분석 |
| **세일즈 파트너** | **+2.5%** | • 고난이도 영업 활동<br>• 신규 고객 발굴 전문성<br>• 매출 직접 기여도 높음 | 신규 영업, 고객 관계 관리 |
| **마케팅 파트너** | **+2.0%** | • 창조적 콘텐츠 제작<br>• 브랜드 가치 창출 기여<br>• 전문 디자인/기획 능력 | 콘텐츠 제작, 캠페인 기획 |
| **교육 파트너** | **+1.8%** | • 지식 전달 전문 능력<br>• 고객 역량 향상 기여<br>• 교육 커리큘럼 개발 | 제품 교육, 사용자 트레이닝 |
| **기술지원 파트너** | **+1.5%** | • 기술적 문제 해결 전문성<br>• 시스템 안정성 기여<br>• 24/7 지원 가능성 | 기술 지원, 문제 해결 |
| **고객서비스 파트너** | **+1.2%** | • 고객 만족도 향상 역할<br>• 관계 유지 및 관리<br>• 대량 처리 업무 특성 | 고객 문의 응대, AS 관리 |

**📊 보너스율 차등화 원칙:**

1. **업무 복잡도 기준:**
   ```
   컨설팅 > 세일즈 > 마케팅 > 교육 > 기술지원 > 고객서비스
   ```

2. **전문성 요구도:**
   - **고도 전문성**: 컨설팅, 세일즈 (2.5% 이상)
   - **중급 전문성**: 마케팅, 교육 (1.8-2.0%)
   - **기본 전문성**: 기술지원, 고객서비스 (1.2-1.5%)

3. **수익 창출 기여도:**
   - **직접 기여**: 세일즈, 컨설팅 (높은 보너스)
   - **간접 기여**: 마케팅, 교육 (중간 보너스)
   - **지원 역할**: 기술지원, 고객서비스 (기본 보너스)

**🎲 실제 적용 예시:**

**방식 1 (성과 연동형) 적용 시:**
```
세일즈 파트너 (목표: 500만원, 보너스: 2.5%)
• 실제 매출 600만원 (목표 달성): 70% + 2.5% = 72.5%
• 실제 매출 400만원 (목표 미달): 70% + 0% = 70.0%

컨설턴트 파트너 (목표: 800만원, 보너스: 3.0%)
• 실제 매출 900만원 (목표 달성): 65% + 3.0% = 68.0%
• 실제 매출 700만원 (목표 미달): 65% + 0% = 65.0%
```

**방식 2 (전문성 인정형) 적용 시:**
```
• 골드 등급(70%) 세일즈 파트너: 70% + 2.5% = 72.5% (매출과 무관)
• 실버 등급(65%) 컨설턴트 파트너: 65% + 3.0% = 68.0% (매출과 무관)
• 플래티넘 등급(75%) 기술지원 파트너: 75% + 1.5% = 76.5% (매출과 무관)
• 브론즈 등급(60%) 마케팅 파트너: 60% + 2.0% = 62.0% (매출과 무관)
```

**🚀 중장기 개선 방안:**
1. **하이브리드 모델 도입**: 기본 보너스 + 성과 추가 보너스


2. **단계별 보너스 시스템**: 목표 달성 정도에 따른 차등 보너스
   ```
   목표 달성률 150% 이상: 타입 보너스율 × 150%
   목표 달성률 100-149%: 타입 보너스율 × 100%
   목표 달성률 80-99%: 타입 보너스율 × 50%
   목표 달성률 80% 미만: 타입 보너스율 × 0%
   ```

- **조직 균형**: 다양한 전문 분야의 균형 잡힌 육성

## ⚠️ 현재 시스템의 문제점

### 1. 중복된 컬럼 구조
- `partner_tiers`와 `partner_users`에 동일한 의미의 컬럼이 중복 존재
- `commission_rate` vs `base_commission_rate` vs `personal_commission_rate`
- 데이터 일관성 관리의 복잡성 증가

### 2. 불명확한 우선순위
- 세 테이블의 설정값이 충돌할 때 어떤 값을 적용할지 불명확
- 개별 설정과 등급 기본값의 관계가 모호

### 3. 미사용 컬럼
- `partner_tiers`의 `base_commission_rate`, `management_bonus_rate`, `discount_rate`가 모두 0.0으로 설정
- 실제 활용되지 않는 컬럼들이 존재

### 4. 복잡한 계산 로직
- 최종 수수료 계산 시 여러 테이블을 참조해야 함
- 성능 및 유지보수성 저하

### 5. 🔥 고정된 목표치의 한계
- **성장성 부재**: 파트너 타입별 고정 목표로 성장 동기 부족
- **개인차 무시**: 경험/능력에 관계없이 동일한 목표 적용
- **시간성 부재**: 연도별/월별 변화하는 시장 상황 미반영
- **등급 연계 부족**: 등급 상승에 따른 목표 조정 시스템 없음

### 6. 🤔 타입 vs 등급별 목표 설정 역할 혼재
- **타입별 목표** (partner_types.target_sales_amount): 전문 분야별 기본 기준
- **등급별 능력**: 개인 성과 및 경험 수준 차별화 부재
- **역할 중복**: 어디서 목표를 관리할지 명확하지 않음

**현재 문제 예시:**
```
브론즈 세일즈 파트너 (신입): 500만원 목표
플래티넘 세일즈 파트너 (베테랑): 동일한 500만원 목표 ❌
→ 등급별 차별화 필요!
```

## 🎯 개선된 동적 목표 시스템 설계

### 1. 🏗️ 새로운 계층 구조 정의

```
Partner Types (기본 최소 기준)
    ↓ 승수 적용
Partner Tiers (등급별 목표 승수)
    ↓ 시간별 조정
Dynamic Targets (연월별 개인 목표)
    ↓ 성과 추적
Performance Tracking (실시간 달성률)
```

### 2. 🎯 타입 vs 등급별 역할 재정의

#### **Partner Types 역할: "최소 기준치 (Baseline)"**
- **목적**: 각 전문 분야별 **최소 요구 성과 기준**
- **특성**: 고정값, 전체 파트너 타입의 **입문 기준**
- **활용**: 신규 파트너 등급 배치 및 최소 성과 검증

```php
partner_types 테이블 역할:
- min_baseline_sales: 해당 타입의 최소 기준 매출 (예: 세일즈 300만원)
- min_baseline_cases: 해당 타입의 최소 처리 건수 (예: 기술지원 50건)
```

#### **Partner Tiers 역할: "등급별 목표 승수 (Multiplier)"**
- **목적**: 등급에 따른 **목표 확장 계수**
- **특성**: 승수값, 경험과 능력에 따른 **목표 스케일링**
- **활용**: 동일 타입 내에서 등급별 차별화된 목표 설정

```php
partner_tiers 테이블 역할:
- sales_target_multiplier: 매출 목표 승수 (브론즈 1.0, 실버 1.5, 골드 2.0, 플래티넘 3.0)
- cases_target_multiplier: 건수 목표 승수
- performance_expectation: 등급별 기대 성과 레벨
```

### 3. 📅 동적 목표 테이블 신설: `partner_dynamic_targets`

```sql
CREATE TABLE partner_dynamic_targets (
    id BIGINT PRIMARY KEY,
    partner_user_id BIGINT, -- 대상 파트너
    target_period_type ENUM('monthly', 'quarterly', 'yearly'), -- 목표 기간
    target_year YEAR, -- 목표 연도
    target_month TINYINT NULL, -- 목표 월 (월별인 경우)
    target_quarter TINYINT NULL, -- 목표 분기 (분기별인 경우)

    -- 매출 목표
    sales_target_amount DECIMAL(15,2), -- 개인별 매출 목표
    team_sales_target DECIMAL(15,2), -- 팀 매출 목표 (관리자인 경우)

    -- 활동 목표
    support_cases_target INT, -- 지원 건수 목표
    new_clients_target INT, -- 신규 고객 획득 목표
    retention_rate_target DECIMAL(5,2), -- 고객 유지율 목표

    -- 목표 설정 정보
    base_multiplier DECIMAL(5,2), -- 등급별 기본 승수
    personal_adjustment DECIMAL(5,2) DEFAULT 1.0, -- 개인별 조정 계수
    market_factor DECIMAL(5,2) DEFAULT 1.0, -- 시장 상황 반영 계수

    -- 성과 추적
    current_achievement DECIMAL(15,2) DEFAULT 0, -- 현재 달성 실적
    achievement_rate DECIMAL(5,2) DEFAULT 0, -- 달성률 (%)

    -- 보상 정보
    base_commission_rate DECIMAL(5,2), -- 기본 수수료율
    bonus_tier JSON, -- 달성률별 보너스 단계
    achieved_bonus_rate DECIMAL(5,2) DEFAULT 0, -- 달성한 보너스율

    -- 메타 정보
    is_active BOOLEAN DEFAULT TRUE,
    created_by BIGINT, -- 목표 설정자
    approved_by BIGINT, -- 승인자
    approved_at TIMESTAMP NULL,

    created_at TIMESTAMP,
    updated_at TIMESTAMP,

    INDEX idx_partner_period (partner_user_id, target_year, target_month),
    INDEX idx_active_targets (is_active, target_period_type),
    FOREIGN KEY (partner_user_id) REFERENCES partner_users(id)
);
```

### 3. 수수료 계산 공식

```
최종 개인 커미션율 = Base Rate + Type Bonus + Tier Bonus + Individual Override
최종 관리 보너스율 = Tier Management Bonus + Individual Override
최종 할인율 = Tier Discount Rate + Individual Override
```

#### 구체적 계산 예시:
- **Base Rate**: 등급별 기본 수수료율 (60-75%)
- **Type Bonus**: 파트너 타입별 추가 보너스 (1.2-3.0%)
- **Tier Bonus**: 등급별 성과 보너스 (현재 미사용)
- **Individual Override**: 개별 파트너별 특별 조정

## 🔧 권장 시스템 구조

### 1. 컬럼 정리 및 통합

#### Partner Types (유지)
- `target_sales_amount` - 타입별 목표 매출
- `commission_bonus_rate` - 타입별 추가 보너스

#### Partner Tiers (수정)
- `commission_rate` → `base_commission_rate`로 통일
- `tier_management_bonus_rate` 추가 (등급별 관리 보너스)
- `tier_discount_rate` 추가 (등급별 기본 할인율)

#### Partner Users (수정)
- `commission_override` - 개별 커미션 조정값 (+ 또는 -)
- `management_bonus_override` - 개별 관리 보너스 조정값
- `discount_rate_override` - 개별 할인율 조정값

### 2. 계산 로직 단순화

```php
// 최종 커미션율 계산
$finalCommissionRate =
    $tier->base_commission_rate +           // 등급 기본율

    ($user->commission_override ?? 0);      // 개별 조정

// 최종 관리 보너스율 계산
$finalManagementRate =
    $tier->tier_management_bonus_rate +     // 등급 관리 보너스
    ($user->management_bonus_override ?? 0); // 개별 조정

// 최종 할인율 계산
$finalDiscountRate =

    ($user->discount_rate_override ?? 0);   // 개별 조정
```
